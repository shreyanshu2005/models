Transaction cost constants — hard-wire these into every label and reward

  • Per-leg STT + brokerage + stamp + charges ≈ 0.11% of trade value. Deduct on BOTH buy and sell legs individually: net_value_buy = price × qty × (1 + 0.0011), net_value_sell = price × qty × (1 − 0.0011). This is the effective round-trip cost of ~0.22%.
  • Short-term capital gains tax (STCG): 20% on net profits realised within the holding period (assumed all positions intraday or <12m for EOD). Apply annually on net realised gains per symbol per year.
  • Benchmark : NIFTY50 from the three benchmark CSVs (2010–2014, 2015–2019, 2020–2026). Buy-and-hold NIFTY from 2016-01-02, apply same STCG on notional annual gains for a fair comparison.

Purging rule (EOD, critical)

Use 30-day purge gap between train end and validation start to prevent label leakage through overlapping feature windows. Walk-forward step = 21 trading days (one month). Embargo = 10 days after each fold end. Validation is always OOS by calendar, never by bar.

§1 Regime detection engine Prereq to all models

A shared regime label is computed daily and injected as a feature/conditioning variable into all three models. This is the HMM + rolling-vol percentile layer from your spec.

HMM 5-State Hidden Markov Model
  • Inputs: NIFTY50 daily log-returns, 20-day realised vol, 10-day momentum Z-score
  • States: Rising, Rally, Sideways, Falling, Crashing
  • Emission: Gaussian diagonal covariance per state
  • Sticky transitions: raised self-transition prior to avoid flickering
  • Trained on NIFTY50 pooled (not per-symbol) — one regime for the entire universe
  • Viterbi decode daily for the current regime label

Library: hmmlearn · Refit: Bimonthly (SLOW) · Transition: FAST

VOL PERCENTILE Rolling volatility band
  • Per-symbol 20-day realised vol, rolling 252-day percentile rank
  • Low: ≤33rd pct → LV
  • Medium: 33–66th pct → MV
  • High: ≥67th pct → HV
  • Updated daily; lookback calibrated quarterly

Feeds all model parameter selectors and SAC position caps

Regime-conditioned model behaviour

Regime HMM State SVM behaviour CNN-LSTM behaviour SAC behaviour
Rising Bull trend Bullish bias, high C Momentum features weighted Positive sizing, low penalty
Rally Fast up-move Buy signals; tighter margin Short lookback patterns Partial position; watch mean-reversion
Sideways Range-bound Neutral / flat signals Mean-reversion features Minimal position; cost wall is binding
Falling Bear trend Sell signals dominate Down-pattern features Short or flat; drawdown penalty raised
Crashing High-vol crash Flat / safety-first Crash-state features, wider window Zero position; max drawdown brake

Implementation: regime as a feature, not a hard switch

Encode the 5-state regime as a one-hot vector (5 dims) + the soft posterior probabilities (5 dims) → 10 additional features appended to every model's feature vector. This is simpler than building separate sub-models per regime while still giving all three models regime-awareness. The SAC reward function also multiplies the drawdown penalty by a regime scalar (1.0 Rising, 1.0 Rally, 1.2 Sideways, 2.0 Falling, 5.0 Crashing).

§2 The three models

Each model is a self-contained module with its own feature pipeline, HP search space, and output adapter. They share the regime feature vector from §1 and the cost constants from §0. All three signals are fused by a simple equal-weight vote → final position decision.

ML · SVM Market trend classification — RBF kernel

Output: {-1 = Sell, 0 = Flat, +1 = Buy} for each symbol each day. Combined with CNN-LSTM and SAC signals.

Feature set (all standardised per symbol, rolling 252-day z-score)

DL · CNN-LSTM Candlestick pattern + sequence learning

Output: Predicted 5-day forward return (regression); thresholded to {-1, 0, +1} signal. Conv layers extract local patterns; LSTM models temporal evolution.

Input tensor shape: (batch, lookback_days, channels)

  • Channels (all vol-normalised): OHLCV (5), SMA10, SMA20, RSI14, ATR14, MACD-diff, Bollinger-%B, OBV-z, volume-z, regime one-hot (5), vol-band one-hot (3) = 22 channels
  • EOD lookback: 30 days (FAST, re-searched quarterly). LC/LV: up to 60d. SC/HV: 20–30d.
  • Target: 5-day forward log-return (net of costs) — winsorise at ±3σ before training

Training protocol

  • Pooled cross-symbol training (stack all 10 symbols) then fine-tune per-symbol for final layer weights
  • Adam optimiser, cosine LR decay, gradient clipping 1.0
  • Early stopping: patience=15 on purged OOS loss
  • Loss: MSE on cost-netted returns. Weight recent observations 2× via sample_weight
  • Refit: Monthly full retrain + weekly fine-tune of Dense head (incremental)
RL · SAC Continuous position sizing

Output: Continuous position weight ∈ [−1, +1] per symbol. Positive = long, negative = short/flat (equity long-only: clip to [0, 1]). SAC's off-policy replay buffer supports incremental intra-month updates.

State space and reward

  • State (per symbol, t): last 20 daily returns, current position, unrealised PnL%, SVM signal, CNN-LSTM return forecast, regime one-hot (5), vol-band one-hot (3), days-since-last-trade, normalised time-in-position → ~35 dims per symbol
  • Action: Δposition ∈ [−1, +1] → clipped to valid range; translated to share count via portfolio fraction
  • Reward: r_t = (pnl_t − tx_cost_t) − λ_dd × max_drawdown_t − λ_turn × |Δpos_t|
  • λ_dd: 0.5 Rising → 2.0 Falling → 5.0 Crashing (regime-scaled)
  • tx_cost_t = price × |Δshares| × 0.0011 on each leg, deducted from reward
  • γ = 0.97–0.99 (EOD), FAST parameter — re-searched quarterly

Simulator requirements (NautilusTrader)

  • Daily fills at close price (EOD, acceptable for daily bar models)
  • 0.11% cost deducted on both legs in the environment step — not as post-hoc adjustment
  • Position caps: RELIANCE/HDFC/BAJ/MARUTI/HERO: up to 30% per name; others: 20%
  • Off-policy: replay buffer accumulates across monthly retrains — carry forward across months

Signal fusion: SVM + CNN-LSTM + SAC → final trade

Simple vote-then-size rule

  • SVM signal : {−1, 0, +1} → direction vote
  • CNN-LSTM signal : thresholded return forecast → {−1, 0, +1} direction vote (threshold = ±0.5% net-of-costs predicted 5d return)
  • SAC action : continuous [0,1] weight → direction sign
  • Fusion: if ≥2 of 3 signals agree on direction → take the trade; position size = SAC weight × regime position cap. If disagreement → flat (0 position).
  • Override: Crashing regime → always flat regardless of signals.

§3 Monthly incremental retrain loop

The continuous learning framework. On the first trading day of each month during the backtest period (Jan 2020 – Jun 2026), this loop fires for all 10 symbols and all 3 models.

01
Regime refit check Monthly trigger
Re-fit HMM on NIFTY50 data up to month-end. Check if any state transition matrix changed by >10% (KL divergence) → if yes, flag all model parameters tagged FAST for mandatory re-search this month. Always re-decode Viterbi path and update regime features for all 10 symbols.
02
SVM: incremental refit Monthly weights
Append the latest month's daily bars to the training set. Retrain SVM (sklearn.svm.SVC) on the full expanding window (Jan 2016 → current month-end). Re-search C and γ via Optuna (30 trials, 5-fold time-series CV with 30-day purge) only if: quarterly trigger has fired, or regime flag from step 01 is set. Otherwise reuse prior HP and just refit weights on new data.
03
CNN-LSTM: full monthly retrain + fine-tune head Monthly weights
Full retrain: pooled cross-symbol, expanding window, early stopping on purged OOS. Then per-symbol head fine-tune: freeze Conv1D + LSTM layers, retrain only the Dense(32) and Dense(1) layers for 5 epochs on the most recent 63 days of that symbol's data (3 months, weighted 2× most recent). HP re-search (Optuna, 20 trials) quarterly or on regime flip for FAST params (lookback, dropout, LR).
04
SAC: experience replay update Weekly+ cadence
SAC is off-policy: the replay buffer accumulates continuously during the backtest month. At month-end: run 10,000 gradient update steps from the buffer. Re-search γ and λ_dd quarterly. Position cap is updated immediately whenever the vol-band classification changes (FAST). Full policy evaluation (Sharpe + drawdown sentinel) on the last 63 trading days to check for drift.
05
Drift sentinel gate Before deployment
For each retrained model: compute rolling 63-day live Sharpe vs backtest Sharpe. If |live − backtest| > 0.5σ OR live hit-rate drops >10pp below backtest hit-rate → trigger mandatory HP re-search regardless of calendar. Gate model deployment: new model only replaces current if OOS Sharpe ≥ 0.0 on the last 63 days. If gate fails → hold prior model, log alert.
06
Cost accounting update
At month-end: compute cumulative realised PnL per symbol. Apply STCG (20%) on net realised gains within the year. Reset intra-year running total for next month. Log: gross PnL, transaction costs deducted, STCG deducted, net PnL, per symbol and portfolio aggregate.

Look-ahead prevention checklist

  • All features computed with data available at market-close t; trade executed at open t+1 (or close t+1 acceptable for EOD)
  • No future returns in any feature; regime decoded at t uses only data up to t
  • CNN-LSTM: causal padding on all Conv1D layers; no BiLSTM on the live path
  • SAC: state at t never includes t+1 fill price
  • Monthly retraining uses data up to last day of prior month only (not current month)

Key strategy events

on_bar Daily trigger
  • Compute all 22 features per symbol
  • Call SVM.predict(features) → direction
  • Call CNNLSTM.predict(sequence) → return_fc
  • Call SAC.act(state) → position_weight
  • Fuse signals → target_position
  • Compute delta vs current_position
  • If |delta| > min_trade threshold → submit order
on_order_filled
  • Record fill price, quantity, time
  • Compute actual tx_cost (fill_price × qty × 0.0011)
  • Update SAC replay buffer with (s, a, r, s') tuple
  • Update running cost accumulator
  • Update realised PnL tracker
on_month_end
  • Run monthly retrain loop (§3)
  • Apply STCG on realised gains
  • Log all metrics to results ledger
  • Check drift sentinel, gate models
  • Swap in retrained models if gate passes

Trade reduction mechanism (1 buy + 1 sell per month target)

Turnover budget per symbol per month

The ensemble fusion rule already suppresses noise (require 2/3 signal agreement). Add an explicit position cooldown : after any trade (buy or sell), lock the symbol for min(5 trading days, next regime change). Combine with SAC's λ_turnover penalty. Target: ≤2 round-trips (4 legs) per symbol per month in Rising/Rally regime, ≤1 in Sideways/Falling. Portfolio-level: with 10 symbols × ≤2 round-trips = ≤20 fills/month → ≤40 leg executions/month → tx_cost ≤ 40 × 0.11% per leg = bounded and budget-tracked.

Metric computation formulas

Exact definitions — implement these in the ledger

  • tx_cost per trade: cost = |shares| × price × 0.0011 — deducted on BOTH buy and sell. Cumulative = Σ all legs.
  • Realised PnL (per position close): pnl = (sell_price − buy_price) × shares − tx_cost_buy − tx_cost_sell
  • STCG (per year): stcg = max(0, net_realised_pnl_this_year × 0.20). Applied end-of-year. Reduces NAV immediately in Dec of each year.
  • Net return (after tx+STCG): r_net = (ending_NAV − beginning_NAV − stcg_paid) / beginning_NAV
  • Sharpe: Sharpe = mean(daily_net_returns × 252) / (std(daily_net_returns) × √252). Use risk-free = 6.5% annualised (Indian 91-day T-bill approx).
  • Beta: β = Cov(strategy_returns, NIFTY50_returns) / Var(NIFTY50_returns) — rolling 252-day window, averaged over backtest.
  • Win rate: n_profitable_round_trips / n_total_round_trips. A round-trip = one buy + corresponding sell.
  • MAPE (CNN-LSTM only): MAPE = mean(|actual_5d_return − predicted_5d_return| / |actual_5d_return|) × 100. Exclude bars where |actual| < 0.1% to avoid division-near-zero.

§7 Build timeline

Week 1–2
Data pipeline: load all 10 CSVs + NIFTY50 benchmarks, clean/align to common trading calendar, build feature matrix (25 features), normalise. Validate no NaN, no future leakage.
Week 3
Regime engine: train 5-state HMM on NIFTY50 (2010–2015 warmup, then 2016–2019 for HP tuning). Build vol-band classifier. Validate regime labels visually against known market events (COVID crash = Crashing, post-COVID rally = Rally).
Week 4–5
SVM initial training: HP search with Optuna on 2016–2017 data, validate on 2018–2019 (purged walk-forward). Record best C, γ per cap segment.
Week 6–7
CNN-LSTM initial training: build (batch, 30, 22) tensors, pooled train on 2016–2017, OOS validation on 2018–2019. HP search for lookback, filters, dropout. Per-symbol head fine-tune.
Week 8–9
SAC initial training: build NautilusTrader gym.Env wrapper, train SAC on simulated 2016–2019 environment, evaluate policy on 2018–2019 hold-out.
Week 10
Fusion + backtest engine: wire up ensemble vote rule, connect to NautilusTrader strategy, build ledger and cost accounting. Smoke-test Jan 2020 – Mar 2020 (COVID crash stress test).
Week 11–12
Full backtest Jan 2020 – Jun 2026: run monthly retrain loop end-to-end. Build drift sentinel. Generate Table 1, Table 2, and cumulative return chart.
Week 13+
Analysis & iteration: analyse per-regime performance breakdown, tune turnover budget, run sensitivity on STCG rate and tx_cost assumptions, write summary report.
